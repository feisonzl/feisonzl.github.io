[原]驱动中ioctl的使用

1 ioctl调用

用户空间使用：
<span class="hljs-keyword">int</span> ioctl(<span class="hljs-keyword">int</span> fd,<span class="hljs-keyword">unsigned</span> <span class="hljs-keyword">long</span> cmd,...);
内核空间声明：
<span class="hljs-keyword">int</span> (*ioctl)(<span class="hljs-keyword">struct</span> inode *inode,<span class="hljs-keyword">struct</span> file *filp,<span class="hljs-keyword">unsigned</span> <span class="hljs-keyword">int</span> cmd,<span class="hljs-keyword">unsigned</span> <span class="hljs-keyword">long</span> arg);`</pre>

### 2 ioctl命令

<pre class="prettyprint">`内核约定了定义ioctl命令的相应方法，详细介绍见Documentation/ioctl/ioctl-number.txt和<span class="hljs-keyword">include</span>/asm-generic/ioctl.h
定义ioctl命令的方法使用了<span class="hljs-number">4</span>个位字段：<span class="hljs-class"><span class="hljs-keyword">type</span>,<span class="hljs-title">number</span>,<span class="hljs-title">direction</span>,<span class="hljs-title">size</span></span>
<span class="hljs-class"><span class="hljs-keyword">type</span>:幻数字段（8<span class="hljs-title">bit</span>）,根据<span class="hljs-title">ioctl</span>-<span class="hljs-title">number</span>.<span class="hljs-title">txt</span>，选择一个号码在整个驱动中使用</span>
number:序号字段（<span class="hljs-number">8</span>bit），表示第几个命令号
direction:传输方向字段（<span class="hljs-number">2</span>bit），表示数据传输方向
size:数据大小字段（<span class="hljs-number">14</span>bit），表示数据传输的大小
命令构造方法：_IOC(direction,<span class="hljs-class"><span class="hljs-keyword">type</span>,<span class="hljs-title">number</span>,<span class="hljs-title">size</span>)</span>
常用构造命令的宏：
#define _IO(<span class="hljs-class"><span class="hljs-keyword">type</span>,<span class="hljs-title">nr</span>)        <span class="hljs-title">_IOC</span>(</span>_IOC_NONE,(<span class="hljs-class"><span class="hljs-keyword">type</span>),(</span>nr),<span class="hljs-number">0</span>)
#define _IOR(<span class="hljs-class"><span class="hljs-keyword">type</span>,<span class="hljs-title">nr</span>,<span class="hljs-title">size</span>)  <span class="hljs-title">_IOC</span>(</span>_IOC_READ,(<span class="hljs-class"><span class="hljs-keyword">type</span>),(</span>nr),(_IOC_TYPECHECK(size)))
#define _IOW(<span class="hljs-class"><span class="hljs-keyword">type</span>,<span class="hljs-title">nr</span>,<span class="hljs-title">size</span>)  <span class="hljs-title">_IOC</span>(</span>_IOC_WRITE,(<span class="hljs-class"><span class="hljs-keyword">type</span>),(</span>nr),(_IOC_TYPECHECK(size)))
#define _IOWR(<span class="hljs-class"><span class="hljs-keyword">type</span>,<span class="hljs-title">nr</span>,<span class="hljs-title">size</span>) <span class="hljs-title">_IOC</span>(</span>_IOC_READ|_IOC_WRITE,(<span class="hljs-class"><span class="hljs-keyword">type</span>),(</span>nr),(_IOC_TYPECHECK(size))
构造命令宏的使用：
#define CMD1 _IO(<span class="hljs-class"><span class="hljs-keyword">type</span>,1)</span>
#define CMD2R _IOR(<span class="hljs-class"><span class="hljs-keyword">type</span>,2,<span class="hljs-title">int</span>)</span>
#define CMD3W _IOW(<span class="hljs-class"><span class="hljs-keyword">type</span>,3,<span class="hljs-title">int</span>)</span>
`</pre>

### 3 预定义命令

<pre class="prettyprint">`预定义命令有三类：
<span class="hljs-number">1.</span>可用于任何文件的命令；
<span class="hljs-number">2.</span>只用于普通文件的命令；
<span class="hljs-number">3.</span>特定于文件系统类型的命令；
用于任何文件的常用预定义命令：
<span class="hljs-number">1.F</span>IOCLEX:设置执行时关闭标志
<span class="hljs-number">2.F</span>IONCLEX:清除执行时关闭标志
<span class="hljs-number">2.F</span>IOASYNC:设置或复位文件异步通知
<span class="hljs-number">2.F</span>IOQSIZE:返回文件或目录大小
<span class="hljs-number">2.F</span>IONBIO:文件IO非阻塞型IO`</pre>

### 4 ioctl参数使用

<pre class="prettyprint">`这里主要讨论ioctl的最后一个参数，该参数常常是指向用户空间的指针，但使用用户空间的指针必须保证合法，否则可能导致内核崩溃或其他安全问题。（copy_from_user和copy_to_user内部已经实现了相关的安全验证，所以不用关心这个问题）
因此，我们需要使用access_ok函数验证地址是否安全：
<span class="hljs-built_in">int</span> access_ok(<span class="hljs-built_in">int</span> <span class="hljs-class"><span class="hljs-keyword">type</span>,<span class="hljs-title">const</span> <span class="hljs-title">void</span> *<span class="hljs-title">addr</span>,<span class="hljs-title">unsigned</span> <span class="hljs-title">long</span> <span class="hljs-title">size</span>);</span>
<span class="hljs-class"><span class="hljs-keyword">type</span>:<span class="hljs-title">VERIFY_READ</span>/<span class="hljs-title">VERIFY_WRITE</span></span>
size:参数大小`</pre>

### 5 ioctl返回值

<pre class="prettyprint">`这里只讨论ioctl命令不匹配时的返回值，通常的做法是返回<span class="hljs-attribute">-EINVAL</span>,但根据POSIX标准，应该返回<span class="hljs-attribute">-ENOTTY</span><span class="hljs-built_in">.</span>
