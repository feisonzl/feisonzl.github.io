由上面的例子看出Device Tree有一下特点：
<span class="hljs-number">1.</span>节点node
    <span class="hljs-number">1.1</span>每个node可以包含sub node和property/value；
    <span class="hljs-number">1.2</span>每个DTS都只有一个ROOT node；
    <span class="hljs-number">1.3</span>一个node可以有多个sub node，但只能有一个<span class="hljs-keyword">parent</span> node；
    <span class="hljs-number">1.4</span>一个node可以包含多个property/value描述该node的具体信息；
    <span class="hljs-number">1.5</span>每个node用节点名字（node name）标识，节点名字的格式是node-name<span class="hljs-variable">@unit</span>-address。如果该node没有reg属性（后面会描述这个property），那么该节点名字中必须不能包括<span class="hljs-variable">@和</span><span class="hljs-keyword">unit</span>-address。
<span class="hljs-number">2.</span>属性/值property/value
    <span class="hljs-number">2.1</span>一般属性：
    属性值表示了该设备节点的具体特性，它的值是多样性的：
    <span class="hljs-number">1</span>)可能为空，即没有值的定义；
    <span class="hljs-number">2</span>)可能为<span class="hljs-number">32</span>bit或<span class="hljs-number">64</span>bit整数值(#<span class="hljs-keyword">size</span>-cells = &lt;<span class="hljs-number">2</span>&gt;)或者数组(reg = &lt;<span class="hljs-number">0x0</span> <span class="hljs-number">0x86c00000</span> <span class="hljs-number">0x0</span> <span class="hljs-number">0x5600000</span>&gt;)；
    <span class="hljs-number">3</span>）可能为<span class="hljs-keyword">string</span>(label = <span class="hljs-string">"cont_splash_mem"</span>)或<span class="hljs-keyword">string</span> list(compatible = <span class="hljs-string">"qcom,msm8953"</span>)；
    <span class="hljs-number">2.2</span>特殊属性：
    #address-cells = &lt;<span class="hljs-number">1</span>&gt;;
    #<span class="hljs-keyword">size</span>-cells = &lt;<span class="hljs-number">0</span>&gt;;
    #类似number，#address-cells表示子节点中reg中的地址元素需要用多少个<span class="hljs-number">32</span>bit值来表示，#<span class="hljs-keyword">size</span>-cells表示子节点中reg中的大小元素需要用多少个<span class="hljs-number">32</span>bit值来表示。`</pre>
